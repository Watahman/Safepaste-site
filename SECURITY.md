# Security policy

## Reporting a vulnerability

Please do not report security vulnerabilities in public GitHub issues and do
not include API keys, tokens, passwords, private keys, personal data, or real
client content in any report.

Use GitHub's private vulnerability-reporting feature for this repository when
it is enabled. If it is not available, contact the repository owner privately
through their GitHub profile and include only a sanitized proof of concept.

## What to include

- A short description of the potential impact.
- Reproduction steps using dummy values only.
- Browser and SafePaste version.
- Any suggested mitigation, if known.

We aim to acknowledge valid reports within seven days and will share progress
privately while the issue is investigated.

## Scope

SafePaste is a local browser extension. Reports involving accidental exposure
of user content, permission misuse, bypassing a block, or unsafe payment
integration are in scope. Do not test against other people's accounts or data.
